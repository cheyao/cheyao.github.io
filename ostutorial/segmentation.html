Segmentation is a way to access more memory developed by Intel. Although it is stupid and everyone hates it, we have to learn it to make our bootloader.

In 16 bit mode, there are 6 registers for segmentation, they are cs (code segment), ds (data segment) es, (extra segment) , ss (stack segment), fs (general purpose F segment) and gs (general purpose G segment).

The value that you are referencing, for example if you use mov [0x1000], 'H' (in nasm, values inside [ and ] are memory positions), you will need to calculate ds * 0x10 + 0x1000, so if you had 0x1000 in ds, you are then referencing the address 0x11000.


PS. You cannot move values straight into the segment registers with mov, you must use a general register as a temporary value holder.
