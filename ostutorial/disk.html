The max size of our OS is 510 bytes at the moment, which is pretty small. We might want to make our OS bigger, so it is essential to be able to read from the disk.

Thankfully, we don't have to deal with turning spinning platters on and off, we can just call some BIOS routines, like we did to print characters on the screen. To do so, we set will use the BIOS function int 0x13, al = 0x02.

AH = 0x02
AL = number of sectors to read (1-128 dec.)
CH = track/cylinder number (0-1023 dec., see below)
CL = sector number (1-17 dec.)
DH = head number (0-15 dec.)
DL = drive number
ES:BX = pointer to buffer


Return:
AH = status
AL = number of sectors read
CF = 0 if successful
   = 1 if error

You can see that it returns CF = 0 if successful, this means that the carry flag is set if something went wrong. The carry flag isn't accessed directly but used as a control structure by other operators, like jc (jump if the carry bit is set)

For the drive number, the BIOS sets dl to our boot drive's number at entry.

Implementation

Lets modify our boot/bootsect.asm to this:

org 0x7C00
bits 16 ; Tell nasm we are in 16 bit mode

SECTORS equ 40 ; A constant, read 40 sectors, should be enough
KERNEL_OFFSET equ 0x7E00 ; Store our other half kernel just after the bootsect

jmp 0:start ; Ignore this, it's just to make sure that the stupid BIOS manufactures doesn't use 0x07C0:0x0000 instead of 0x0000:0x7C00
start:
    mov [DRIVE], dl ; Saves the boot drive number
    mov ah, 0x02 ; Function number
    mov al, SECTORS ; Saves the boot drive number
    mov ch, 0x00 ; Cylinder 0
    mov cl, 0x02 ; Start reading from sector 2, sector 1 is out boot sector
    mov dh, 0x00 ; Head 0
    mov bx, KERNEL_OFFSET ; Put the read sectors here
    int 0x13 ; Call

    jc disk_error ; Jump to disk error if carry bit is set

    cmp al, SECTORS ; Check if the right amount of sectors is read
    jne sectors_error ; Jump if not equal

    jmp KERNEL_OFFSET ; Now if there is no errors, jump there

disk_error:
    mov di, DISK_ERROR
    call puts
    jmp $

sectors_error:
    mov di, SECTORS_ERROR
    call puts
    jmp $

puts: ; Our print function
    push eax ; Store the registers
    push di
    mov ah, 0x0E
.loop:
    mov al, [di] ; Store the stuff in memory position edi to al
    cmp al, 0
    je end ; If al is equal to 0, goto end
    int 0x10
    inc di
    jmp .loop
end:
    pop di
    pop eax
    ret

DRIVE: db 0
DISK_ERROR: db "Disk read error", 0
SECTORS_ERROR: db "Incorrect number of sectors read", 0

times 510-($-$$) db 0 ; Padding and magic number
dw 0xaa55
; Now since we are at address 512, we are in the second sector
; Note: You shouldn't typically do this, this will only work if you put the code right after the sector above
; I am only doing this to simplify this lesson
    mov di, HELLO
    call puts
    jmp $

HELLO: db "Hello world from another sector!", 0

Now recompile this, and boot it using the command qemu-system-x86_64 -fda bootsect.bin note that I've added the -fda flag, this means that the bootsect.bin should be a floppy disk
