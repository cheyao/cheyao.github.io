When the computer boots, it will first run the BIOS, then the BIOS will load the first sector of the disk, called the bootsector, into the RAM address 0x7C00 (This number is in hex).

To make sure that the disk is bootable, BIOS checks that bytes 511 and 512 of the boot sector are bytes 0xAA55 (the magic number)

Creating the bootsector

Create a directory called boot and a file inside it called bootsector.asm and paste the following code into it:

org 0x7C00
jmp $

times 510 - ($ - $$) db 0 ; Fill with 510 zeros minus the size of the previous code
dw 0xaa55 ; The magic number

org 0x7C00 tells nasm that out code will be placed at 0x7C00

The jmp is a computer instruction (here is a list of instructions) that tells the cpu to jump to the memory position specified after it, so for example jmp 5 is to tell the cpu to go to the memory position 5, and start executing from there. $ is a special token, and nasm will replace it with the current position, so the code jmp $ means to jump to the current position, which means the cpu will go into a dead loop.

The $$ in times 510 - ($ - $$) db 0 is also a special token, which means the start position of the code, this code lets nasm to fill the rest of the file with zeros.

The dw 0xaa55 is the magic number, to let the BIOS know that this disk is bootable

Testing the code

Let's test our code!

Since booting on a real computer is very inconvenient and you probably don't have a spare computer (like me), we will use a emulator.

First, we need to compile the assembly into binary codes (or called opcodes), to do this, we'll use the command nasm -o bootsect.bin boot/bootsect.asm, paste it into the terminal (while you are inside the project's root directory).

Now we'll emulate the computer with qemu-system-x86_64 bootsect.bin, you should see a window appear that has a blinking cursor on it (like this), that means we have successfully made a bootable disk!
