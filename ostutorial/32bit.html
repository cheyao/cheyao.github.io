Until now our OS was in 16 bit mode or called Real mode. Real mode limits our registers to 16 bit values, and uses segmentation, so we can only store values up to 65535, and we can only access a little over 1 MB of memory. We might want to use more memory and store bigger values, so we would need to switch to 32 bit protected mode (or even 64 bit mode, but it's way too complex for now).

A20 pin

The A20 pin is a feature that nobody wants, CPU manufactures continues to produce it. It is a feature that will enable/disable the 0x100000 address bit, If we don't enable it, we would only be able to address odd address bits (Well expect on virtual machines and a small amount of real machines, they enable it by default)

We enable it like this:

in al, 0x92
test al, 2
jnz .after
or al, 2
and al, 0xFE
out 0x92, al
.after:

Note: this code doesn't work on all computers, see here on how to enable it reliably.

Global Descriptor Table

After the A20 pin is enabled we can load the gdt. The gdt is a binary data structure telling the CPU about memory segments. We will define the easiest gdt possible - one that has only 2 segments.

GDT:
    .Null:
        dq 0
    .Code:
        dw 0xffff    ; segment limit, bits 0-15
        dw 0x0000    ; segment base, bits 0-15
        db 0x00      ; segment base, bits 16-23
        db 10011010b ; flags (8 bits)
        db 11001111b ; flags (4 bits) + limit, bits 16-19
        db 0x00      ; segment base, bits 24-31
    .Data:
        dw 0xffff
        dw 0x0000
        db 0x00
        db 10010010b
        db 11001111b
        db 0x00
    .Pointer:
        dw GDT.Pointer - GDT - 1
        dd GDT

This is the gdt, you can see what flags I used from here.

Now we need to load the gdt and go to protected mode, we can do it like this:

GDT_DATA equ GDT.Data - GDT
GDT_CODE equ GDT.Code - GDT

    cli ; Clear interrupts
    lgdt [GDT.Pointer] ; load gdt
    mov eax, cr0
    or eax, 1 ; Set protected mode enable bit
    mov cr0, eax
    jmp GDT_CODE:init_pm ; Far jump - automatically sets cs to the segment it's jumping to

bits 32
init_pm:
    mov ax, GDT_DATA ; Load all segments except cs with data segment
    mov ds, ax
    mov ss, ax
    mov es, ax
    mov fs, ax
    mov gs, ax

Now we are in 32 bit mode, but all the interrupts are broken, we will learn how to print onto the screen in 32 bit mode in the next chapter.
